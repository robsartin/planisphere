# Plan 05 — Satellites Design Spec

**Date:** 2026-04-16
**Status:** Approved
**Depends on:** Plan 02 (Astro Core) — complete

## 1. Purpose

Render ~200 visually observable satellites at correct positions on the sky dome, propagated via satellite.js (SGP4) from TLE data. Each satellite shows as a small colored dot with a short ground track trail (past 2 minutes). Hovering shows name, NORAD ID, orbital altitude, and velocity. TLE data fetched at runtime from CelesTrak with bundled fallback.

## 2. Scope

### In scope

- satellite.js integration for SGP4/SDP4 TLE propagation.
- TLE data: runtime fetch from CelesTrak (visual satellites group, ~200 objects) with bundled fallback.
- Build script to generate bundled TLE snapshot (`data/tle/visual.txt`).
- TLE 3-line format parsing into satellite records.
- Per-satellite propagation: ECI → geodetic → observer-relative Alt/Az.
- Horizon filtering (above horizon only).
- Ground track trail: propagate at 10-second intervals over the past 2 minutes (12 trail points per satellite).
- Billboard rendering for satellite positions (small colored dots, distinct from star white).
- Polyline rendering for ground track trails.
- Tooltip integration: hover shows satellite name, NORAD ID, orbital altitude (km), velocity (km/s).
- ADR for satellite.js.

### Out of scope

- Multiple TLE group selection (Plan 06 UI).
- Web Worker offload for propagation.
- Satellite pass prediction / future track projection.
- Flare prediction (Iridium, etc.).
- Real-time animation / continuous propagation (Plan 06 time scrubber drives updates).

## 4. Data

### Bundled TLE snapshot

`data/tle/visual.txt` — CelesTrak visual satellites in 3-line TLE format. Generated by `scripts/build-tle.mjs` which fetches from `https://celestrak.org/NORAD/elements/gp.php?GROUP=visual&FORMAT=tle`. ~200 satellites, ~30KB.

### Runtime fetch

At app startup, attempt to fetch fresh TLE data from the same CelesTrak URL. On success, use fresh data. On failure (CORS block, network error, timeout), fall back silently to the bundled snapshot.

## 6. Key types

### SatelliteRecord (parsed from TLE, stable)

```ts
type SatelliteRecord = {
  readonly name: string;
  readonly noradId: number;
  readonly satrec: SatRec; // satellite.js internal propagation record
};
```

### VisibleSatellite (computed per frame, ephemeral)

```ts
type VisibleSatellite = {
  readonly name: string;
  readonly noradId: number;
  readonly alt: number; // altitude above horizon, degrees
  readonly az: number; // azimuth, degrees
  readonly height: number; // orbital altitude, km
  readonly velocity: number; // orbital velocity, km/s
  readonly trail: readonly { readonly alt: number; readonly az: number }[];
};
```

### Error types

```ts
type TleParseError = { kind: "tle-parse-failed"; message: string };
type TleFetchError = { kind: "tle-fetch-failed"; message: string };
```

## 7. Data flow

1. `sat/fetch.ts` attempts runtime TLE fetch from CelesTrak. On failure, reads bundled `data/tle/visual.txt`.
2. `sat/tle.ts` parses the 3-line TLE text into `SatelliteRecord[]` via `satellite.twoline2satrec`.
3. `sat/propagate.ts` propagates each satellite at the given time:
   - `satellite.propagate(satrec, date)` → ECI position + velocity.
   - `satellite.eciToGeodetic(positionEci, gmst)` → geodetic (lat/lon/alt).
   - `satellite.ecfToLookAngles(observerGd, positionEcf)` → elevation + azimuth from observer.
   - Filter to elevation > 0 (above horizon).
   - Compute orbital velocity from ECI velocity vector magnitude.
   - Compute trail: propagate at 10-second intervals over the past 2 minutes (12 trail points), each converted to observer Alt/Az.
4. `scene/satellites.ts` renders billboards at current positions + polylines for trails.
5. Tooltip picks satellites same as stars/bodies (billboard `id` carries `VisibleSatellite`).

## 8. Visual properties

### Satellite dots

- Color: #00FF88 (bright green) — distinct from white stars and colored planets.
- Size: 6px billboard.
- Sprite: small filled circle (similar to planet sprite generator but green).

### Ground track trails

- Color: #00FF88, alpha 0.3 (faded green).
- Width: 1px polyline.
- 12 points over 2 minutes (10-second intervals).

## 9. Satellite-to-AltAz conversion

satellite.js provides ECI (Earth-Centered Inertial) coordinates. The conversion chain:

1. `satellite.propagate(satrec, date)` → `{ position: EciVec3, velocity: EciVec3 }`.
2. Compute GMST: `satellite.gstime(date)`.
3. ECI to geodetic: `satellite.eciToGeodetic(position, gmst)` → `{ longitude, latitude, height }` (radians, km).
4. Observer geodetic: `{ longitude: lonRad, latitude: latRad, height: 0 }`.
5. ECI to ECF: `satellite.eciToEcf(position, gmst)`.
6. Look angles: `satellite.ecfToLookAngles(observerGd, positionEcf)` → `{ azimuth, elevation, rangeSat }` (radians).
7. Convert to degrees: `elevation * 180/PI`, `azimuth * 180/PI`.

## 10. Tooltip integration

Add `isVisibleSatellite` type guard (checks for `noradId` field). Format:

- Name + NORAD ID
- Altitude above horizon (degrees)
- Azimuth (degrees)
- Orbital altitude (km)
- Velocity (km/s)

## 11. Testing strategy

### `src/sat/tle.ts` — TDD, ≥ 90% coverage

- Parse valid 3-line TLE block → correct name, NORAD ID, satrec.
- Parse multiple satellites from concatenated TLE text.
- Reject malformed TLE (missing lines, bad checksum).
- Return Result error on empty input.

### `src/sat/propagate.ts` — TDD, ≥ 90% coverage

- ISS TLE at a known time → orbital altitude ~400km.
- Observer at known location → look angles produce alt/az in valid ranges.
- Satellite below horizon → excluded.
- Trail has 12 points.
- Velocity is positive and reasonable (~7 km/s for LEO).

### `src/sat/fetch.ts` — mocked fetch, ≥ 90% coverage

- Fetch succeeds → returns fresh TLE text.
- Fetch fails → returns bundled fallback text.
- Fetch times out → returns bundled fallback.

### `src/scene/satellites.ts` — mocked Cesium, ≥ 80% coverage

- N satellites → N billboards + N polylines.
- Empty input → nothing rendered.
- Update clears previous.

## 12. Error handling

- **TLE fetch failure:** Non-fatal. Falls back to bundled data. Logged as warning.
- **TLE parse failure:** `Result<SatelliteRecord[], TleParseError>`. Non-fatal to app — stars/bodies/constellations still render.
- **Propagation failure:** Individual satellites with bad TLEs (decayed orbits, etc.) silently skipped. `satellite.propagate` returns `false` for position on failure — check and skip.
- **All satellite failures are non-fatal.** The sky view works without satellites.
